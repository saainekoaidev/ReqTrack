# 0004 ガント列項目と Excel(.xlsx)エクスポート方式

## Status

Accepted

## Context

[0003](0003-ce2-aligned-estimation-wbs.md) で見積・WBS・スケジュールを ce2 準拠にする方針を決めた。ce2 の Excel(openpyxl)出力は **WBS シート + スケジュール(ガント)シート** の 2 面構成で、MS Project mpp の代替として顧客に渡せる品質を持つ。ReqTrack でも同等の列項目と、見積諸元・見積根拠を明記した Excel 出力(US-015 / US-016)を提供する。

ce2 のシート構成(ce2 スクリプト `gen_wbs_issue.py` / `gen_wbs_fu2026.py` より):
- **WBS シート**: `No / 機能 / 対象 / 作業タスク / 工程 / 工数(人日) / 担当者 / 工数推定理由` + 末尾「効率化調整」行 + 担当者別工数サマリ。工程ごとに行の背景色。
- **スケジュールシート**: 固定列 `No / フェーズ / 対象 / 作業タスク / 工数 / 開始日 / 終了日 / 担当者` + **3 段日付ヘッダ(月 / 日+曜日 / コマ①②③④)** + 工程色のバー塗り + 休日列の淡色 + 凡例。

## Decision

### 画面ガント(US-015)
Web のガントを ce2 準拠の列項目に再構成する。
- 固定列: No / フェーズ(工程) / 対象 / 作業タスク / 工数(人日) / 開始 / 終了 / 担当者。
- 日付軸: 月ヘッダ / 日+曜日ヘッダ / 4 コマ(①②③④)の 3 段。土日祝の列は淡色。
- バーは工程色で、コマ単位の開始〜終了に描画。進捗(US-008)は塗りで重畳。効率化調整はバー描画せず別掲。
- 凡例(工程色)と担当者別工数サマリを併記。

### Excel エクスポート(US-016)
- ライブラリは **exceljs**(backend / Node)。セル結合・背景色・3 段ヘッダ・複数シートを再現できる。
- エンドポイント `GET /api/projects/:id/estimate.xlsx` で .xlsx をストリーム返却。
- シート構成:
  1. **見積**: 見積諸元(最小単位 0.25 人日 / 4 コマ・コマ時間帯 / レビュー率・下限 / 効率化調整 / 営業日定義 / 開始日)+ 工程別・担当者別の工数サマリ + 合計。
  2. **WBS**: 上記 WBS シートと同一列(見積根拠 = 工数推定理由 列を含む)+ 効率化調整行。
  3. **ガント**: 上記スケジュールシート(3 段ヘッダ + 工程色バー)。
- 数値の正本は DB/ドメイン関数。Excel は表示専用ビューとし、計算は backend ドメインで一元化する。

## Consequences

- ce2 と同じ列・見方の Excel を ReqTrack 単体で生成でき、見積諸元と見積根拠が 1 ファイルで完結する。
- exceljs を backend 依存に追加。バイナリ生成のため API はファイルストリームを返す(フロントはダウンロードリンク)。
- mpp(MS Project)出力は本 ADR の範囲外(ce2 は mpxj、ReqTrack は将来課題)。当面は .xlsx を mpp の代替とする。
- ガント列・Excel 列は ce2 を一次参照とし、差異が生じたら本 ADR に追記して同期する。
