# 0012 AI見積の確認クッションと担当割付のタイミング

## Status

Accepted

## Context

US-037 で「AIで要件からガントまで生成」を 1 アクション化したところ、見積内容(工数・根拠)を確認しないままガント画面へ直行してしまい、「根拠を見て補正してから計画を確定する」という実務の流れに合わなかった。あわせて「WBS に担当者を割り付けるタイミングが不明確」という指摘があった。

## Decision

- **AI見積は実行直後にガントへ遷移しない。** ステップ3(見積・ガント)で「AIで見積を生成」すると、見積結果(機能/対象/作業, 工数, 稼働率, 見積根拠, 担当)を同じ画面の表に展開する。
- 表で **工数・稼働率・見積根拠(estimateNote)・担当(assignee)** を確認・補正できる。**担当割付はこの見積確認の段で行う**(AI見積は未割当で生成 → 確認時に割付。レビューは PL を自動割付、ガント生成後も管理画面で変更可)。
- 「**この見積でガントを作成 →**」の明示操作を経て初めて `scheduleTasks` でガントを生成し、進捗管理へ遷移する。
- 直行用の `POST /api/projects/:id/ai-plan`(estimate+schedule)は UI からは使わない(API としては残置)。AIを使う流れは必ず「AI見積 → 確認 → 承認」を経由する。

## Consequences

- 見積根拠を確認・補正してから計画を確定でき、誤った見積のままガント化されない。
- 担当割付の場所が「見積確認ステップ(と手組みWBS、管理画面)」に明確化。
- ステップ3のクリック数は増えるが、実務の意思決定(承認)を 1 操作として明示できる。
