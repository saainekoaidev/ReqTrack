# 0008 UI シェル刷新(入口分離 / テーマ / アイコン / ステッパー / 設定タブ)

## Status

Accepted (supersedes 一部 of [ui.md] 初期レイアウト方針)

## Context

ユーザより画面デザインの全面的な改善要望:
1. 新規作成系と進捗管理系で入口を分ける。
2. 新規作成系: ペイン分割/ヘッダでワークフロー的に現在位置が分かる。
3. 進捗管理系: ペイン分割/ヘッダ、右上で参照プロジェクト選択、左ペインに各機能への遷移メニュー。
4. 入口部分に全体設定画面(タブ: 基本設定 / 要員登録 / 休日登録)。
5. 全体に淡い緑系統、グラフィカルフォント、ピクトグラム的アイコン(Windows 標準でない)。見映えと分かりやすさ重視(ガントは難しいと言われるため)。

## Decision

### ルーティング/シェル構成
- **ランディング `/`**: 大きな入口カード「新規作成」「進捗管理」「設定」。
- **新規作成系 `/create/*`**: 上部に**ステッパー(新規→取込/WBS→見積→ガント)**で現在位置表示。配下に既存の new/import/wbs/requirements/tasks/estimate を配置。
- **進捗管理系 `/manage/*`**: 共通シェル。**ヘッダ右上に参照プロジェクト選択(全体で共有=ProjectContext)**、**左ペインに遷移メニュー**(ガント/進捗報告/日報/遅延)。配下に gantt/reports/daily/delays。
- **設定 `/settings`**: タブ(基本設定 / 要員 / 休日)。

旧フラットな nav(全機能横並び)は廃止し、モード別シェルへ。既存ページコンポーネントは再利用し、ラッパー(レイアウト)を差し替える。

### テーマ / アイコン / フォント
- **配色**: 淡い緑系統。token を更新(primary=緑, bg=淡緑, surface=白, アクセント)。
- **フォント**: グラフィカルな丸ゴシック(`M PLUS Rounded 1c`, Google Fonts)。読込不可時はシステムフォントへフォールバック。
- **アイコン**: インライン SVG のピクトグラム(自前 `Icon` コンポーネント、Windows shell アイコン不使用)。
- デザインシステムの一次正は引き続き `frontend/src/styles/app.css`([CLAUDE.md] 準拠)。

### プロジェクト選択の共有
進捗管理系は `ProjectContext`(選択中 projectId を保持)を導入し、各画面はそれを参照。URL の `?projectId=` とも整合させる。

## Consequences

- 目的(作る/管理する/設定する)で画面が分かれ、現在位置と遷移が明快になる。
- 進捗管理系はプロジェクトを 1 か所で選べ、各機能がそれに追従。
- 既存ページは中身を保ちつつレイアウトを再編(ルート変更あり)。E2E/テストの導線アサーションを更新する。
- フォント/アイコンは外部 CDN フォント + 自前 SVG。オフライン時はフォントのみ degrade(機能影響なし)。
