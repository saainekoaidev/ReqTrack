# 0014 ガントの小数日対応・連続時間軸描画と WBS ツリー表示

## Status

Accepted (US-015 の Excel 風コマ表示・US-004 の整数日スケジューリングを置き換える)

## Context

従来のガントは ce2 のスケジュール表(Excel)に倣い、(1)工数 ÷ 稼働率を**切り上げて整数営業日**にし、(2)1 日 = 24px の**コマ(セル)グリッド**にバーを整数列で配置していた。これは「ce2 はスケジュール作成処理の参考」という意図に反し、0.5 人日のタスクが 1 日に膨らむなど不正確だった。また WBS の親行(機能/対象)が表示されず、1.1 から始まり折り畳みもできなかった。

## Decision

- **スケジューリングは小数日のまま**(`scheduleTasks`, ADR の `workingDaysNeeded = 工数 ÷ 稼働率`、切り上げない)。稼働日(土日祝を除く)の勤務時間内に、前タスクの終了**時刻**から次タスクを**連続**割り付ける(隙間・コマ無し)。`plannedStart/End` は時刻まで保持(既定 始業 9:00 / 1日=設定の作業時間)。
- **描画は稼働時間軸(working-time)**。土日祝を圧縮した連続軸上に、バーを `開始/終了の稼働日位置(小数)` の割合(%)で配置する。セルグリッドは持たない。軸の日目盛り・年/月ヘッダは**年込み**で表示。
- **WBS は 3 階層ツリー**(機能→対象→作業)を行表示し、親行は配下の葉から 開始/終了/工数 を集計。**折り畳み(展開/収納)**に対応。
- **ガントは右ペインの全幅**を使う(`shell-content--wide`)。表示しきれない場合のみ横スクロール(各稼働日 最小 34px)。

## Consequences

- 0.5 人日 → 半日バー、1.5 人日 → 1 日+半日、と小数日が正確に反映される。コマ起因の歪みが無い。
- 親行の集計バー(サマリ)で機能/対象単位の期間も俯瞰でき、折り畳みで大規模 WBS も扱える。
- `plannedStart/End` が時刻を持つため、遅延検出(線形補間)もより細かくなる。
- Excel 出力(US-016)は別途整合を取る(本 ADR の対象外。必要なら追従)。
