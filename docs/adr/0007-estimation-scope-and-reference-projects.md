# 0007 見積スコープと参照資料プロジェクト(新規/既存)

## Status

Accepted

## Context

ユーザより見積に関する 2 点:
1. 見積の対象範囲は **システム(アプリケーション)構築部分に限定**でよい。
2. 新規開発なら動作環境の想定から構築工数を見積れるが、**既存システム改修**では既存の設計/ソース等の参照資料なしに見積れない。よって「改修対象の参照資料群をフォルダにまとめ、ReqTrack に『見積可能プロジェクト』として登録 → 隙間時間で内容を読取り(インデックス化) → 見積依頼時に新規/既存を選び、既存なら登録済みプロジェクトを選択して配下資料を見積の参照に含める」仕組みが要る。

現状の ReqTrack は LLM 非依存(決定論的)。「資料を読んで AI が工数を出す」最終ステップは本来 LLM を要する。

## Decision

### 見積スコープ = システム構築部分
標準工程は `基本設計 / 詳細設計 / コーディング / 単体テスト / 結合テスト`([0003](0003-ce2-aligned-estimation-wbs.md))で、これがシステム構築の範囲。要件管理・運用・インフラ構築等は当面スコープ外(必要なら別 ADR)。本スコープを画面/Excel/ドキュメントに明示する。

### 新規 / 既存 区分と参照資料プロジェクト
- `ReferenceProject`(参照資料プロジェクト)を新設: `name`, `rootPath`(資料フォルダ), `note`, スキャン結果(`ReferenceFile`: 相対パス, サイズ, 抽出テキスト抜粋, 種別)。
- **スキャン(読取り)**: rootPath 配下を走査し、テキスト系(.md/.txt/.csv/.json/ソースコード等)は冒頭抜粋を、その他はメタのみ登録。隙間時間=任意タイミングで再スキャン可能。
- **見積依頼時**: 新規プロジェクトで「新規 / 既存」を選択。既存なら登録済み `ReferenceProject` を選び、`Project.referenceProjectId` で関連付け。見積/取込の参照先として配下資料を提示・利用できるようにする。

### AI 生成見積の位置づけ(現状の割り切り)
資料内容から工数を自動算出する最終ステップは **LLM 連携の将来拡張点**とする。現状は:
- 参照資料の登録・スキャン・関連付け・閲覧までを提供(フレームワーク)。
- 数値見積は既存の決定論的手段(標準工程展開 + 手入力/取込)で行い、既存案件では参照資料を見ながら人/将来AIが精緻化する。

LLM を導入する際は本 ADR を Superseded で更新し、`ReferenceFile` の抜粋＋要件を文脈に工数を生成する。

## Consequences

- 既存改修でも「参照資料を ReqTrack 管理下に置き、見積時に参照する」運用が成立する。
- 完全自動の数値見積は LLM 導入までは未提供(割り切り)。フレームワークは先に用意しておく。
- スキャンはローカルファイル読取り。大容量/バイナリは抜粋せずメタのみ(性能・容量の割り切り)。
