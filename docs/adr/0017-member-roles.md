# 0017 要員の役割(プロジェクト管理者/チームリーダー/担当者)

## Status

Accepted (US-005 の自由記述 role を役割選択に整理)

## Context

従来 要員の role は自由記述で、誰が作業者になれて誰がレビュワーになれるかが定義されていなかった。レビュー自動割付も role==='PL' 固定だった。実務では「プロジェクト管理者・チームリーダー・担当者」を区別し、作業者資格・レビュワー資格を役割から導きたい。

## Decision

要員の役割を 3 つから選択(`Member.role`):

- `pm` プロジェクト管理者
- `leader` チームリーダー
- `member` 担当者(既定)

役割から資格を導出(`frontend/src/lib/roles.ts`):

- **レビュワー資格** = `pm` または `leader`。
- **作業者資格**(ガント上のタスク担当) = `member` または `leader`。`pm` は原則作業者にならない。
  - フォールバック: 作業者資格者が 1 人も居なければ全員を候補にする(1 人運用=PM が兼務するケースを許容)。
- プロジェクト管理者はチームリーダーを兼務しうる(役割は代表値を 1 つ持つ。兼務は運用で表現)。
- **レビュワー自動割付**は最上位(`pm`)を優先し、居なければ `leader`(旧データの 'PL' も互換で拾う)。

担当(assignee)選択 UI は作業者候補のみを出す。要員一覧は役割ラベルを表示する。

## Consequences

- 作業者/レビュワーが役割から一貫して決まり、レビュー自動展開(US-014/US-044)の割付先が明確になる。
- 既存の自由記述 role(例 'PL')は作業者候補から外れる場合があるが、開発データのため許容(必要なら再設定)。
- 「レビュー工程の有無選択」「複数レビュワーの振り分け」は US-044 で扱う。
