# 0006 柔軟な取込とプロジェクト起点

## Status

Accepted

## Context

新規プロジェクトの導線として「見積から」「ガントから」の 2 起点を設けたい(US-020)。
- 見積から: 要件一覧 or 見積明細を取り込んで開始(US-019)
- ガントから: 空のガントに階層タスクを直接編集して開始(US-018)

取込について、ユーザは「テンプレートを固定したくない。テンプレがあれば精度は上がるが、**自由体裁のファイルや自然文の入力でも(精度はかなり落ちてよいので)見積・ガントのタスクへ展開**できてほしい」と要望。ReqTrack は LLM 非依存(決定論的)である。

## Decision

### 取込(US-019): テンプレ優先 + 自由体裁/自然文フォールバック(ヒューリスティック、LLM 非依存)
入力種別を 3 つ受け付ける:
1. **自然文テキスト**: 改行 / 「・-*」箇条書き / 句点「。」で分割し、各セグメントを 1 要件とする。
2. **ファイル(xlsx / csv)**:
   - **要件取込**: ヘッダに `要件 / 内容 / 改修 / 概要 / content` 等を含む列があればその列を要件本文に採用(テンプレ＝高精度)。無ければ **各行で最長の文字列セルを本文に採用**(自由体裁＝低精度)。
   - **見積明細取込**: ヘッダ `WBS / タスク / 作業 / 工程 / 工数(人日) / 稼働率 / 担当 / 見積根拠` 等のエイリアスでマッピングし、タスクを直接生成。マッピングできない列は無視。
3. 取り込んだ要件は、必要に応じて標準工程展開(US-013)で見積・ガントのタスクへ広げる。

ヘッダ照合は**別名(エイリアス)辞書**で行い、列順・表記揺れにある程度耐える。**精度が出ない入力は「それなりに」取り込む**(ユーザ了承済み)。xlsx 解析は既存依存の **exceljs** を読取りに使う。

### ガント編集(US-018): WBS 表のインライン階層編集
ガント/WBS の一覧表上で、行=タスクを直接編集する。
- 機能(level1)追加 / 子タスク追加(level+1)/ 行削除 / セル編集(名称・工程・工数・稼働率・担当)。
- wbsId は追加時に自動採番(level1=連番、子=親 wbsId + "." + 兄弟連番)。
- 既存 API(POST/PATCH/DELETE tasks)を拡張(taskInput に level/parentId/wbsId/phase を追加)。

### プロジェクト起点(US-020)
「新規プロジェクト」画面で名称入力 + 起点選択。
- 見積から → 取込画面(US-019)へ
- ガントから → 空プロジェクト作成 → WBS 編集(US-018)へ

## Consequences

- 定型に縛られず着手でき、テンプレ準拠時は精度が上がる二段構え。
- 自由体裁/自然文は誤抽出が起こりうる(設計上の割り切り)。取込後は WBS 編集(US-018)で手直しできるため実用に足る。
- LLM 連携(高精度抽出)は将来の拡張余地。導入時は本 ADR を Superseded で更新する。
- 取込は exceljs(読取)+ CSV 簡易パーサ。大容量や複雑な結合セルは非対応(割り切り)。
