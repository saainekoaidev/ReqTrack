# 0020 明示的なタスク間依存(前提タスク)

## Status

Accepted ([0015] で将来課題としていた明示的タスク間リンクを導入)

## Context

これまでの依存は「同一対象(親)配下の工程順」という暗黙のものだけで、機能をまたぐ任意の先行関係(例: 2.3 の前提が 1.1)を表現できなかった。これでは実質ウォーターフォール固定で、ガントとして不十分。

## Decision

- **`TaskDependency` モデル**を追加(`taskId`=後続, `predecessorId`=前提, ユニーク制約)。1 タスクに複数の前提を設定可能。
- **API**: `GET /api/tasks` は各タスクに `predecessorIds` を含める。`PUT /api/tasks/:id/predecessors { predecessorIds }` で置換(自己参照除去・**循環依存は 400 で拒否**)。
- **スケジューラ**: 前提を辺とする**トポロジカル順**でタスクを配置し、各タスクの開始 = max(開始日, 同一対象の工程順起点, **全前提の終了**, 要員の空き)。区間ギャップ充填(US-047)と併用。循環が残った場合は元順で末尾補完(安全弁)。
- **UI**: WBS編集の葉タスク行に「前提タスク」列。現在の前提をチップ表示(×で解除)、セレクトで他の葉タスクを前提として追加。

## Consequences

- 機能をまたぐ任意の先行関係を表現でき、ネットワーク型(非ウォーターフォール)の計画が組める。
- 前提は葉タスク間で有効(親は集約のため計画日を持たない)。
- 暗黙の工程順依存(同一対象)と明示依存は併存し、開始は両者の遅い方に従う。
- 前提終了後に配置されるため、要員の前詰め(US-047)と整合する。
