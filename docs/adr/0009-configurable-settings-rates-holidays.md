# 0009 設定の可変化 / 要員単価 / 祝日自動取得 / 初版生成の配置 / UI 文言

## Status

Accepted

## Context

ユーザ指摘(まとめ):
1. 祝日をカレンダー(日本の祝日)から自動取得したい。
2. 各機能から最初の画面へ「戻るボタン」で帰りたい(システム名クリックのみは分かりにくい)。
3. 基本設定を固定値でなく**範囲制限つきの任意値**にしたい。**見積の最小単位は 0.25 → 0.1 人日**。限界値は適切に。
4. 要員に**単価(円/時)**を登録し、人日(8h/日)から工賃を概算したい。
5. 「ガント初版を作成」ボタンが無反応な理由と、ガント画面冒頭にある不自然さ。
6. 画面文言から内部番号(US-xxx)を消す(説明文は残す)。

## Decision

### 設定の可変化 (US-027)
`Settings` を単一行(シングルトン)で新設し、範囲制限つきで編集可能にする。既定と範囲は下記:

| 設定 | 既定 | 範囲 |
|------|------|------|
| `hoursPerDay`(1日の作業時間) | 8 | 1〜24 |
| `minEstimateDays`(見積の最小単位/丸め) | **0.1** | 0.01〜1 |
| `reviewRatio`(レビュー率) | 0.3 | 0〜1 |
| `reviewMinDays`(レビュー下限) | **0.1** | 0〜5 |
| `defaultUtilization`(既定稼働率) | 1.0 | 0.05〜1 |

API `GET/PUT /api/settings`。範囲外は 400(zod)。レビュー自動展開(US-014)は定数でなく Settings の ratio/min を使う。見積入力の step/丸めは `minEstimateDays` を使う。基本設定タブは閲覧専用表示から**編集フォーム**へ。

### 要員単価 (US-028)
`Member.hourlyRate`(円/時, 任意, 0 以上)を追加。工賃概算 = 人日 × `hoursPerDay` × 単価。要員登録フォームに単価欄、ガント担当者別工数サマリ等に概算工賃を併記。

### 祝日自動取得 (US-025)
日本の祝日 API(`https://holidays-jp.github.io/api/v1/<year>/date.json`)を backend から取得し一括登録。`POST /api/holidays/import?year=YYYY`。ネットワーク不可/プロキシ時はエラーメッセージを返す(割り切り)。重複日はスキップ。

### 初版生成の配置 (US-029)
ガント初版生成は**新規作成フロー**(見積の次)に「ガント初版を生成」アクションとして配置。**進捗管理>ガント**画面のボタンは「スケジュールを再生成」に改称し、結果(生成件数/対象0件)をメッセージ表示。無反応の原因(工数0タスクのみ→対象0件)はフィードバックで解消。

### 戻るボタン / UI 文言 (US-026)
各シェルのヘッダに「ホーム」ボタンを追加(ブランドクリック以外の導線)。進捗管理系/新規作成系のヘッダ右に配置。画面に表示する文言から `US-xxx` 表記を除去(説明文は残す)。**今後も UI 文言に内部番号を出さない**。

## Consequences

- 設定が案件ごとに調整可能になり、見積最小 0.1 や柔らかいレビュー下限に対応。
- 祝日取得は外部 API 依存(オフライン/プロキシ環境では失敗しうる→手入力併用)。
- 単価により工数に加えコスト概算が見える。
- 初版生成の意味と位置が明確になり、無反応問題も解消。
- 内部番号は仕様書(requirements.md/ADR)にのみ残し、UI からは排除。
